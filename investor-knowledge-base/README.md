## 🧭 知识库定位

### 核心定位
**用系统架构思维构建投资人知识库**  
- **不是**：零散的投资笔记
- **而是**：可演化、可检索、可复用的投资认知操作系统

### 差异化
| 维度 | 传统投资人 | 本知识库 |
|--|--|--|
| 知识管理 | 散乱笔记 / PDF收藏夹 | 结构化图谱 + SOP-20 检索 |
| 案例分析 | 经验主义 / 零散复盘 | 模式提炼 + 知识复用 |
| 决策支持 | 直觉判断 | 数据驱动 + 认知校准 |

## 🏗️ 知识库架构

### 一级分类
```
investor-knowledge-base/
├── 1-investment-philosophy/       # 投资哲学（价值投资/成长投资/套利）
├── 2-industry-analysis/           # 行业研究（AI/区块链/硬件/量子计算）
├── 3-company-analysis/            # 公司分析（财务/团队/技术/市场）
├── 4-decision-frameworks/         # 决策框架（风险评估/估值模型/投后管理）
├── 5-learning-system/             # 学习系统（SOP-20阅读/案例库/认知校准）
└── 6-notes/                       # 个人笔记（会议记录/读书笔记/行业观察）
```

### 核心原则
1. **Everything is Data**：所有知识必须数字化、可检索
2. **Pattern Over Detail**：提取模式而非细节（SOP-20）
3. **Cognitive Alignment**：与AI co-pilot 形成认知对齐
4. **Feedback Loop**：投资结果反馈 → 知识库迭代
