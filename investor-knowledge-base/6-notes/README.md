## 📊 笔记管理原则

### 1. Everything is Data
- 所有笔记必须数字化（Obsidian / Roam Research）
- 所有笔记必须可检索（关键词、标签、链接）
- 所有笔记必须可连接（双向链接、知识图谱）

### 2. Pattern Over Detail
- 提取模式而非细节（SOP-20）
- 重点关注"为什么"而非"是什么"
- 建立模式之间的关联

### 3. Feedback Loop
- 每周回顾：检查笔记完整性、准确性
- 每月复盘：提炼模式、更新知识图谱
- 每季度升级：优化笔记结构、方法论

## 🔄 笔记更新流程

### Step 1: 笔记创建
- 选择笔记类型（会议/读书/行业）
- 填充核心要素（时间、地点、参与者）
- 标记关键信息（重点、疑问、行动）

### Step 2: 笔记整理
- 提取模式（SOP-20第2步）
- 构建知识图谱（Obsidian/Roam链接）
- 添加元数据（领域、难度、优先级）

### Step 3: 笔记应用
- 实战测试（SOP-20第4步）
- 更新笔记（根据实战反馈）
- 知识复用（跨项目、跨领域）

### Step 4: 笔记归档
- 季度归档：整理、优化、升级
- 年度归档：沉淀、传承、分享
- 知识输出：文章、课程、演讲

## 🔄 知识复利机制

### SOP-20 阅读流程
1. **有目的阅读**（20%时间第一遍）
2. **提取模式**（而非细节）
3. **构建知识图谱**（而非笔记）
4. **实战测试**（每3天）

### 知识复利公式
```
知识复利 = (输入 × 理解) × (输出 × 反馈) × (迭代 × 复用)
```
